Week 2: First Assignment Reflections

This week I completed and submitted the first assignment. The process revealed several insights about my understanding of the material and areas where I need to focus more attention.

Assignment Overview

The assignment required implementing a basic algorithm and analyzing its performance characteristics. While the implementation itself was straightforward, the analysis portion challenged me to think more deeply about computational complexity.

What Went Well

  • Code structure: Clean, modular implementation with good documentation
  • Testing: Comprehensive test cases covering edge cases
  • Time management: Started early and finished before the deadline

Challenges Encountered

The most significant challenge was understanding the time complexity analysis. Initially, I misidentified the dominant term in the algorithm's runtime. After revisiting the lecture notes and working through examples, the concept became clearer.

Example Code Snippet

function analyze(data) {
  // O(n log n) sorting step
  data.sort((a, b) => a - b);
  
  // O(n) linear scan
  for (let i = 0; i < data.length; i++) {
    console.log(data[i]);
  }
  
  // Overall: O(n log n)
}

Key Takeaways

Understanding the theoretical underpinnings is just as important as writing code that works.

Moving forward, I'll pay more attention to:

  1. Breaking down algorithms into their constituent parts
  2. Identifying nested loops and recursive calls
  3. Considering best-case, average-case, and worst-case scenarios

Looking Ahead

Next week's material builds on these foundations, introducing more advanced data structures. I'm excited to see how these concepts connect and compound.
